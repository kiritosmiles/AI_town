# AI Town — 产品需求文档 (PRD)

> 版本: 1.0 | 日期: 2026-06-26 | 状态: Design

---

## 1. 产品概述

### 1.1 产品定位

AI Town 是一款 **AI 原生的单机生活模拟游戏**，通过 Godot 4 引擎呈现 RPG Maker 风格的 2D 俯视世界。每个角色（玩家 + NPC）由独立的大型语言模型 (LLM) agent 驱动，拥有分层记忆、自我反思、技能学习和社交能力。玩家通过编写 prompt 创建角色，在 1000×1000 的 Tile 世界中体验从出生到死亡的一生。

### 1.2 核心差异化

| 特性 | 说明 |
|---|---|
| **AI 灵魂** | 每个行为由 LLM 实时决策，非脚本驱动 |
| **分层记忆** | Agent 拥有工作记忆+情景记忆+语义记忆+遗忘曲线，接近人类记忆模型 |
| **GraphRAG 知识图谱** | 五类实体（角色/地点/组织/事件/物品）入图，驱动 NPC 关系推理 |
| **真实社会涌现** | NPC 之间有独立的社会网络，法律由玩家议会+NPC共同演化 |
| **透明 AI** | 玩家可查看 Agent 感知→规划→决策→后果的完整思维链 |
| **人机协作** | 玩家设定高层目标，Agent 自主规划执行，可随时介入 |

### 1.3 发布平台

- **主平台**: Steam (Windows / macOS / Linux)
- **分发方式**: 买断制（暂定）
- **引擎**: Godot 4 (GDScript) + Python Agent 服务（本地进程）

### 1.4 参考竞品

- AI Town (a16z) — 浏览器端 AI agent 小镇模拟
- AIvilization — LLM 驱动的文明演化模拟
- Generative Agents (Stanford) — 学术原型，记忆流+反思+规划
- Project Sid — 游戏化 AI agent 社会模拟

详见 [竞品分析文档](2026-06-26-ai-town-competitive-analysis.md)。

## 2. 用户画像

### 2.1 画像 A：人生模拟党（核心画像）

**一句话**: "我想活出另一个人生。"

| 维度 | 描述 |
|---|---|
| **动机** | 体验一个不同于现实的虚拟人生，观察 AI 角色的自主成长 |
| **核心行为** | Prompt 精心设计角色背景 → 观察 Agent 如何决策 → 与 NPC 自然社交 → 组建家庭 → 走完一生 |
| **乐趣来源** | Agent 的拟人化程度、记忆连续性、情感表达的自然度、角色的独特性 |
| **厌恶** | NPC "失忆"、对话模板化、角色行为不合性格、无法影响世界 |
| **关键需求** | Agent 意识面板、分层记忆可视化、自然语言对话、子嗣继承 |
| **Steam 游戏对标** | 《模拟人生》系列、《星露谷物语》、《环世界》 |

### 2.2 画像 B：建造经营党

**一句话**: "我要成为镇上首富。"

| 维度 | 描述 |
|---|---|
| **动机** | 在 AI 驱动的经济系统中积累财富和资源，追求排名登顶 |
| **核心行为** | 买地炒地皮 → 盖楼开店 → 优化产业链 → 盯排行榜 → 超越 NPC 首富 |
| **乐趣来源** | 经济策略有效性、资产增长曲线、排行榜排名上升、地产扩张 |
| **厌恶** | 经济系统有漏洞、竞争无目标、排行榜无挑战性、建造无成就反馈 |
| **关键需求** | 排行榜（资产榜+声望榜+技能榜）、经济数据面板、资产总览、建造蓝图 |
| **Steam 游戏对标** | 《过山车大亨》、《城市天际线》、《房产达人》 |

### 2.3 画像 C：社会实验观察者

**一句话**: "我想看这个世界怎么演化。"

| 维度 | 描述 |
|---|---|
| **动机** | 观察 LLM agent 如何自发形成社会网络、法律、文化和经济秩序 |
| **核心行为** | 观察 NPC 社会网络 → 追踪特定 NPC 一生轨迹 → 分析法律演化 → 研究经济周期 |
| **乐趣来源** | 社会涌现现象的发现、Agent 行为模式分析、长期演化的不可预测性 |
| **厌恶** | NPC 行为重复、社会无演化、无法查看全局数据 |
| **关键需求** | GraphRAG 全局浏览器、NPC 决策日志、社会演化时间线、数据分析导出 |
| **Steam 游戏对标** | 《矮人要塞》、《WorldBox》、《物种起源》 |

## 3. 核心用户旅程

### 3.1 画像 A 典型旅程

```
开局配置 → 引导问答+自由prompt创建角色(小明)
  │
  ▼
进入世界 → 小明在小镇家中醒来
  │         Agent 感知: "天亮了，饿了，今天没有安排"
  │         玩家设目标: "找到自己的人生方向"
  │
  ▼
探索期 → 在镇上逛，认识 NPC 老王(咖啡店老板)
  │        聊天自然，老王记住小明
  │        学会基础社交和移动
  │
  ▼
成长期 → 在咖啡店打工，学烹饪技能
  │        跟老王关系加深（好感度上升）
  │        参加烹饪考试→获得证书
  │
  ▼
成熟期 → 攒够钱买地开面包店
  │        与 NPC 小美恋爱→结婚→生子
  │        管理店铺+家庭平衡
  │
  ▼
暮年期 → 子嗣成年，可选择传承
  │        身体衰老，回顾一生
  │        自然死亡或事件死亡
  │
  ▼
终局 → 结算成就积分
  │    ├─ 有子嗣: 继承子嗣继续 / 重开（子嗣留在世界当NPC）
  │    └─ 无子嗣: 只能重开
  │
  ▼
新循环 → 继承子嗣的人生，或全新角色
```

### 3.2 画像 B 典型旅程

```
开局配置 → 引导问答（偏商业倾向性格）
  │
  ▼
进入世界 → 分析本地经济（价格面板）
  │         发现商业街人流大，餐饮空缺
  │
  ▼
原始积累 → 打工攒启动资金
  │         研究材料价格波动
  │         自学烹饪技能
  │
  ▼
创业期 → 低价买入商业街 Plot
  │       建造小型咖啡店（蓝图）
  │       雇佣 NPC 做店员
  │       盯排行榜 → 当前排名第 47
  │
  ▼
扩张期 → 利润再投资买地
  │       开设连锁店
  │       炒地皮低买高卖
  │       排行榜 → 进入前 10
  │
  ▼
登顶 → 成为镇上首富
  │     资产超越 NPC 前首富
  │     排行榜 #1
```

### 3.3 画像 C 典型旅程

```
开局配置 → 创建观察者角色（低调性格）
  │
  ▼
进入世界 → 打开 GraphRAG 全局浏览器
  │         查看完整实体关系图
  │         标记关键 NPC 追踪
  │
  ▼
观察期 → 追踪 NPC 老王的一生决策
  │       观察法律如何从"无酒令"→"允许酒类销售"
  │       记录经济危机事件链
  │       导出数据做分析
  │
  ▼
干预实验 → 故意在议会推动一条法律
  │         观察社会反馈和连锁反应
  │         对比干预前后社会网络变化
```

## 5. 功能需求详细说明（核心功能节选）

### 5.1 角色创建 (P0-1)

**流程**:

```
Step 1: 引导问答
  ├─ 性别: 男 / 女
  ├─ 年龄段: 18-25 / 26-35 / 36-45 / 46-60
  ├─ 性格倾向: 外向开朗 / 内向安静 / 理性思考 / 感性直觉 / 冒险进取 / 稳健保守
  └─ 梦想职业: 经商 / 工匠 / 学者 / 从政 / 自由职业 / 未决定

Step 2: 自由 Prompt
  ┌────────────────────────────────────────────┐
  │ 请用 50-500 字描述你的角色:                   │
  │                                              │
  │ 我叫陈小明，22岁，从小在小镇长大。父亲是木匠，  │
  │ 母亲是小学老师。我性格内向但做事认真，一直梦想   │
  │ 开一家自己的咖啡店。高中毕业后在家具厂打工攒钱，  │
  │ 但心里一直没忘记那个咖啡店的梦想...             │
  └────────────────────────────────────────────┘

Step 3: 系统生成
  → LLM 解析 prompt → 生成角色属性 + 性格参数 + 初始记忆
  → 角色在预设小镇的家中出生(创建)
  
  生成结果示例:
    姓名: 陈小明      年龄: 22
    健康: 85          体力: 90
    心情: 70          饥饿: 50
    性格: 内向(I0.7) 尽责(C0.8) 宜人(A0.65)
    技能: 木工基础Lv1(受父亲影响)
    初始记忆: [小时候在爸爸木工坊帮忙的记忆...]
```

### 5.2 Agent 决策循环 (P0-4)

```
每个决策周期:

  ┌─────────────────────────────────────────┐
  │  1. 感知 (Perceive)                      │
  │     世界状态摘要 → Agent 工作记忆          │
  │     "我在咖啡店,老王在吧台后,天气晴,       │
  │      饥饿45/100, 存款3200元"              │
  └──────────────┬──────────────────────────┘
                 ▼
  ┌─────────────────────────────────────────┐
  │  2. 检索 (Retrieve)                      │
  │     情景记忆检索(相关记忆Top-K)            │
  │     GraphRAG查询(自身相关实体关系)          │
  │     语义记忆加载(技能/信念/知识)            │
  └──────────────┬──────────────────────────┘
                 ▼
  ┌─────────────────────────────────────────┐
  │  3. 决策 (Decide)                        │
  │     LLM 综合:感知+记忆+用户目标+性格       │
  │     输出: 下一步行动(Tool调用 或 自然语言)   │
  └──────────────┬──────────────────────────┘
                 ▼
  ┌─────────────────────────────────────────┐
  │  4. 执行 (Execute)                       │
  │     Godot 引擎执行 Tool → 改变世界状态     │
  │     结果返回 Agent                        │
  └──────────────┬──────────────────────────┘
                 ▼
  ┌─────────────────────────────────────────┐
  │  5. 记录 (Record)                        │
  │     新经历写入情景记忆流(含重要性评分)      │
  │     更新 GraphRAG 图谱(如有新实体/关系)     │
  │     检查是否需要触发反思                    │
  └─────────────────────────────────────────┘
```

### 5.3 分层记忆系统 (P0-5 + P1-4 + P1-5)

```
Agent 记忆架构:

  ┌─────────────────────────────────────────┐
  │  工作记忆 (Working Memory)                │
  │  · 当前上下文窗口                          │
  │  · 实时世界感知                            │
  │  · 最近N条交互记录                         │
  │  · 当前短期目标                             │
  └──────────────┬──────────────────────────┘
                 │ 每次交互后自动写入
  ┌──────────────▼──────────────────────────┐
  │  情景记忆 (Episodic Memory)               │
  │  · 时间戳 + 摘要 + 重要性评分(1-10)        │
  │  · 向量嵌入存储(本地向量数据库)             │
  │  · 遗忘曲线: 可检索度 = f(时间, 重要性, 检索) │
  │  · 遗忘阈值以下 → 归档（深度检索可恢复）     │
  └──────────────┬──────────────────────────┘
                 │ 每N条新记忆触发反思
  ┌──────────────▼──────────────────────────┐
  │  语义记忆 (Semantic Memory)               │
  │  · 技能: {cook_latte_art: Lv.2}          │
  │  · 信念: {"老王是好人": 置信度0.8}         │
  │  · 知识: {"咖啡店周二客流量最低": 来源记忆}  │
  │  · 洞察: {"我发现拉花的关键是手腕角度"}     │
  └─────────────────────────────────────────┘

遗忘曲线设计:
  
  可检索度 = e^(-λt) × w_importance × (1 + α × n_retrievals)
  
  其中:
    t = 距现在的时间
    w_importance = 重要性评分 1-10 映射的权重 (1→0.3, 5→0.6, 10→1.0)
    n_retrievals = 被其他记忆检索引用的次数
    λ = 衰减速率 (重要性越高, λ越小)
    α = 检索强化系数
  
  遗忘阈值: 可检索度 < 0.15 → 归档
```

### 5.4 GraphRAG 知识图谱 (P0-6 + P1-6)

**五类实体**:

| 实体类型 | 示例 | 属性 |
|---|---|---|
| 角色 | 小明、老王、小美 | 姓名/年龄/职业/性格 |
| 地点 | 老王咖啡店、镇政府 | 坐标/PlotID/建筑类型/容量 |
| 组织 | 小镇商会、议会 | 成员列表/成立时间/宗旨 |
| 事件 | 2024-03-15抢劫案 | 时间/地点/参与者/结果 |
| 物品 | 小明的咖啡机、某幅画 | 所有者/位置/价值/历史 |

**关系类型**: 血缘、婚姻、师徒、朋友、同事、雇佣、邻居、仇敌、所有、位于、参与、见证...

**三层访问**:

| 层级 | 访问者 | 范围 | 用途 |
|---|---|---|---|
| Agent 内部查询 | NPC/玩家 Agent | 只能查自己相关的关系+记忆 | 决策驱动 |
| 玩家记忆查询 | 玩家 | GraphRAG 局部视图（Agent视角） | 记忆可视化参考 |
| 全局浏览器 | 社会观察者 | 完整图谱+变化历史+关系展开 | 外部观察/调试工具 |

### 5.5 NPC 心跳模式 (P0-12 + P1-7 + P2-9)

开局可选三种模式，影响 LLM 调用量和世界真实度:

| 模式 | 行为 | LLM 调用 | 推荐配置 |
|---|---|---|---|
| **C (经济)** | 默认。NPC按作息表移动，仅在玩家互动时触发完整Agent决策。互动时回填"这段时间发生了什么" | 最低（仅互动时） | 低配机器/想省API费 |
| **B (均衡)** | 玩家半径(视野+紧密关系者)内NPC完整Agent决策，范围外作息表 | 中等 | 推荐默认 |
| **A (沉浸)** | 全量心跳:所有NPC每N分钟(游戏时间)跑完整Agent决策 | 最高 | 高配机器/观察者画像 |

### 5.6 建造系统 (P1-13)

**流程**:

```
1. 浏览未售 Plot
     │
2. 购买 Plot → 扣除金钱，Plot 归属变更
     │
3. 选择建筑蓝图
     ├─ 住宅: 小木屋/公寓楼/别墅
     ├─ 商业: 咖啡店/餐厅/商店/写字楼
     ├─ 工业: 工厂/仓库/工坊
     └─ 公共: 学校/医院/公园（需议会批准）
     │
4. 确认建造
     ├─ 需求检查: 金钱 + 材料(木材×N,石料×N...)
     └─ 不满足 → 材料不足提示
     │
5. 工期倒计时
     例: 小木屋 2月(游戏时间) ≈ 8现实小时
     │
6. 建造完成 → 建筑出现在 Plot 内的 Tile 上
     │
     ├─ 可进入/使用/经营
     └─ 影响周边地产价值
```

### 5.7 生命与死亡 (P1-14 + P2-4/5/6)

**寿命计算**:

```
基线寿命 = random(60, 90) 岁
    
修正因子:
  × 生涯健康系数 (0.7 - 1.3)     ← 长期健康管理
  × 医学发展系数 (1.0 - 1.5)     ← 社会医学研发水平
  × 事件影响 (±5-10岁)           ← 重大疾病/事故

最终寿命 = 基线 × f(修正因子)
```

**死亡流程**:

```
角色死亡
  │
  ├─ 死因判定
  │    ├─ 自然死亡 (达到寿命上限)
  │    └─ 事件死亡 (疾病/事故/破产/他杀)
  │
  ├─ 子嗣检查
  │    ├─ 有成年子嗣(≥18岁)
  │    │    ├─ [继承子嗣] → 切换到子嗣视角继续游戏
  │    │    └─ [结束重开] → 结算积分 → 子嗣留在世界当NPC
  │    │                    → 创建全新角色 (与新角色无关)
  │    │
  │    └─ 无成年子嗣
  │         └─ [结束重开] → 结算积分 → 创建全新角色
  │
  └─ 成就积分结算
       ├─ 财富积分: 峰值资产排名
       ├─ 技能积分: 技能树完成度
       ├─ 社会积分: 声望/影响力/人际关系广度
       └─ 传承积分: 子嗣数量+质量
            │
            └─ 用于新角色初始属性有限度优化
```

## 11. 风险与缓解

| 风险 | 影响 | 缓解 |
|---|---|---|
| LLM API 响应慢 | Agent 卡顿 | 本地超时降级→预设行为回退；本地模型选项 |
| LLM 幻觉导致 NPC 行为异常 | 世界失真 | 关键行为(交易/建造)走Go引擎验证，不纯信LLM |
| 记忆无限膨胀 | 性能下降 | 遗忘曲线+归档机制；玩家可调记忆保存时长 |
| GraphRAG 查询复杂度过高 | 延迟超标 | 限制Agent查询深度(2跳)；外部工具无限制 |
| 单机暂停导致游戏时间不连续 | 经济周期异常 | 不上线=世界暂停，无"追赶"问题 |
| Godot GDScript 性能瓶颈 | 掉帧 | Tile分块加载；大量NPC渲染用对象池 |

## 13. 竞品借鉴追溯

PRD 中各项设计从竞品分析文档的具体借鉴来源：

### 13.1 借鉴 vs 反推创新

| 类型 | 数量 | 定义 |
|---|---|---|
| **直接借鉴** | 4 项 | 竞品做得好的，我们复用 |
| **竞品缺失反推** | 7 项 | 竞品没有的空白 = 我们的差异化空间 |
| **竞品启发升级** | 4 项 | 竞品做了但不够深，我们深化 |

### 13.2 全部借鉴映射

| PRD 章节 | PRD 内容 | 来源 | 类型 |
|---|---|---|---|
| §5.2 Agent 决策循环 | 感知→检索→决策→执行→记录 五步循环 | **Generative Agents (Stanford)** — 记忆流+检索三因子+反思+规划 | 直接借鉴 |
| §5.3 分层记忆 | 工作记忆→情景记忆→语义记忆 三层架构 | **Generative Agents** 记忆流为基础，+ MemGPT/Letta 分层思想 | 直接借鉴 |
| §5.3 检索三因子 | 近因+相关性+重要性评分 | **Generative Agents (Stanford)** 论文原文 Retrieval 算法 | 直接借鉴 |
| §5.3 反思提炼 | 每N条新记忆触发反思→生成洞察 | **Generative Agents (Stanford)** 的 Reflection 机制 | 直接借鉴 |
| §5.3 遗忘曲线 | 可检索度 = e^(-λt) × w × (1 + α × n) | **竞品缺失反推** — 所有竞品均无遗忘机制，记忆无限膨胀 | 缺失反推 |
| §5.4 GraphRAG 知识图谱 | 五类实体(角色/地点/组织/事件/物品)入图 | **竞品缺失反推** — 四个竞品横比 GraphRAG 列全部 ❌ | 缺失反推 |
| §5.5 NPC 心跳 C/B/A 三模式 | 从按需触发到全量心跳的渐进 | **Project Sid** 大规模 Agent 并行 + **AIvilization** 文明级调度 | 启发升级 |
| §3.2 画像 B 建造经营党 | 排行榜竞争、超越 NPC 首富 | **AIvilization** 的文明间竞争排名 | 启发升级 |
| §3.2 画像 C 社会观察者 | GraphRAG 全局浏览器、社会演化时间线 | **AI Town (a16z)** 的对话日志可视化 → 扩展到图谱级别 | 启发升级 |
| §1.2 核心差异化：Agent 意识透明化 | 🧠感知→📋规划→⚡决策→后果 面板 | **AI Town (a16z)** 的 NPC 对话日志 → 升级为结构化思维链 | 启发升级 |
| §5.1 Prompt 创建角色 | 引导问答 + 自由 prompt | **竞品缺失反推** — 全部竞品无玩家角色创建 | 缺失反推 |
| §4.2/P2 技能树 | 烹饪/建造/驾驶/医术 职业技能树 | **AIvilization** 的科技树 → 从文明级降至个体级 | 启发升级 |
| §4.4/P2 事件驱动死亡 | 疾病/事故/破产/他杀 | **AIvilization** 的文明级事件 → 转为个体死亡事件 | 启发升级 |
| §5.6 建造系统 | Plot 地产 + 蓝图建造 | **Project Sid** 的物品/建造系统证明 Agent 建造可行 | 直接借鉴 |
| §4.4/P2 子嗣继承 | 死亡→继承子嗣→成就积分 | **竞品缺失反推** — 四个竞品均无生命循环 | 缺失反推 |
| §4.4/P2 法律治理 | NPC 宪法底线 + 玩家议会 | **竞品缺失反推** — 四个竞品均无法律系统 | 缺失反推 |

> **核心发现**: 约 60% 的核心设计来自竞品分析的"缺失反推"——不是抄竞品做了什么，而是找到他们没做什么。
