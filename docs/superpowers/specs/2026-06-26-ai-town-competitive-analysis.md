## 3. 详细分析

### 3.1 AI Town (a16z)

#### 项目定位

AI Town 是 a16z 开源的浏览器端演示项目，展示 LLM agent 在一个俯视 2D 小镇中的自主行为。定位是**技术 Demo + 开发者游乐场**，非商业游戏产品。

#### 技术架构

| 层 | 技术 |
|---|---|
| 前端渲染 | PixiJS (2D 渲染) |
| 后端 | Convex (实时数据库) |
| Agent 引擎 | TypeScript，基于 Generative Agents 论文 |
| LLM | OpenAI GPT 系列 |
| 记忆 | 向量检索（简单） |

#### 世界设计

- 地图: 单屏固定小镇（~30×20 Tile 量级），无法扩展
- NPC: 预设 ~10 个，无动态生成
- 时间: 游戏内循环推进
- 无地产/建造系统

#### 角色设计

- Agent 决策: 感知→检索→反思→规划→行动（Generative Agents 模型）
- 自定义: 开发者修改代码配置，非玩家创建
- 记忆: 简单记忆流，无分层，无遗忘
- 无 Tool 学习/技能系统

#### 社会模拟

- NPC 之间有基础交互（聊天），但无深度关系系统
- 无经济/法律/建造
- 社会涌现程度: **低**（NPC 数量少，交互浅）

#### 可视化

- 2D 俯视小镇，像素风格
- NPC 以简单精灵移动
- 无 Agent 意识面板
- **亮点**: 所有 NPC 对话日志可查看

#### 优势 & 我们可以借鉴

| 优势 | 借鉴 |
|---|---|
| 像素俯视风格清新可爱 | 美术方向参考 |
| 对话日志可视化 | Agent 意识面板的"行动后果"区域 |
| 开源代码结构清晰 | Agent 循环参考 |
| 社区活跃，有二次开发 | 可复用社区资产(Convex 集成经验) |

#### 劣势 & 我们如何超越

| 劣势 | AI Town 方案 |
|---|---|
| NPC 数量少(10个)，社会太单薄 | **1500+ NPC**，城镇级社会 |
| 地图小且不可扩展 | **1000×1000 Tile**，可生长世界 |
| 无玩家角色创建 | **Prompt 创建角色**，人机协作 |
| 记忆简单，NPC 容易"失忆" | **分层记忆+遗忘曲线** |
| 无经济/建造/法律/继承 | **完整社会系统** |
| 浏览器端性能受限 | **Godot 4 原生引擎 + Steam** |
| 非商业产品 | **全功能游戏产品** |

### 3.2 Generative Agents (Stanford)

#### 项目定位

斯坦福 HCI 组 2023 年发表的学术论文《Generative Agents: Interactive Simulacra of Human Behavior》。定义了 Agent 记忆流+反思+规划的经典架构。定位是**学术研究**，验证 LLM agent 能否产生可信的人类行为模拟。

#### 技术架构

| 层 | 技术 |
|---|---|
| 引擎 | Phaser.js |
| Agent | 记忆流 → 检索(Recency+Relevance+Importance) → 反思 → 规划 |
| LLM | OpenAI GPT-3.5/4 |
| 记忆 | 自然语言记忆流，向量检索 |

#### 世界设计

- 地图: 单屏小镇 Smallville，固定布局
- NPC: 25 个，每个有预设背景故事(一段话)
- 时间: 游戏内模拟时间推进
- 无扩展性

#### 角色设计（核心贡献）

**记忆流 (Memory Stream)**:
```
每个 Agent 的所有经历以自然语言形式存储在记忆流中:
  "2023-02-14 09:00: 在咖啡店遇到了 Isabella，聊了情人节计划"
  "2023-02-14 12:00: 在公园散步，看到学生们在准备情人节活动"
  ...
```

**检索三因子**:
1. **Recency** (近因): 最近发生的记忆权重高
2. **Relevance** (相关性): 与当前情境相关的记忆权重高
3. **Importance** (重要性): LLM 评分 1-10，重要事件权重高

**反思 (Reflection)**:
```
每 N 条新记忆触发反思:
  Prompt: "根据以上 5 条记忆，关于 Klaus 的社交模式，你能得出什么洞察？"
  
  生成: "Klaus 最近经常在咖啡店社交，他可能在寻找浪漫关系"
  → 这条反思写入记忆流，成为更高级的"元记忆"
```

**规划 (Planning)**:
```
每天开始生成当日计划:
  "8:00 起床 → 9:00 去咖啡店 → 12:00 午餐 → 14:00 公园散步 → 18:00 回家"
```

#### 社会模拟

- 论文核心实验: 举办情人节派对——NPC 自主传播消息、策划、参加
- NPC 之间形成自发社交网络
- **涌现程度**: 中（25 个 NPC，实验场景）
- 无经济/法律/建造系统

#### 优势 & 我们可以借鉴

| 优势 | 借鉴 |
|---|---|
| **记忆流+检索三因子** | 情景记忆基础架构直接复用 |
| **反思提炼洞察** | P1-3 反思提炼的核心参考 |
| **规划生成** | Agent 的📋规划模块 |
| **重要性评分** | 遗忘曲线的 w_importance 参数 |
| **可验证的涌现行为** | 作为测试场景的参考（派对组织） |

#### 劣势 & 我们如何超越

| 劣势 | AI Town 方案 |
|---|---|
| 仅 25 个 NPC，学术规模 | **1500+ NPC** |
| 记忆无限增长，无遗忘 | **遗忘曲线，归档机制** |
| 反思粗粒度，仅文本洞察 | **反思→自动解锁技能/更新信念** |
| 无结构化技能/知识 | **语义记忆层（技能/信念/知识）** |
| 无玩家角色，纯观察 | **Prompt 创建+人机协作** |
| 无 GraphRAG | **五类实体知识图谱** |
| 嵌入式模拟，不可分发 | **Steam 可分发产品** |

### 3.3 AIvilization

#### 项目定位

AIvilization 是一个 LLM 驱动的文明演化模拟器。玩家可以选择一个文明并引导其发展，NPC 文明由 LLM agent 驱动进行外交、战争、科技研发。定位是**策略模拟游戏**，侧重宏观文明而非个体生活。

#### 技术架构

- 平台: 浏览器端
- LLM: 多种 API 支持
- Agent: 文明级 Agent（非个体）

#### 世界设计

- 地图: 六角格地图，可扩展
- 文明: 多个 AI 文明并行演化
- 时间: 回合制，跨千年尺度

#### 角色设计

- **无个体 Agent**: 文明作为整体决策单元
- 无个体记忆/技能/成长
- 无玩家角色创建（玩家选择文明）

#### 社会模拟

| 系统 | 说明 |
|---|---|
| 外交 | 文明间结盟/宣战/贸易 |
| 科技 | 科技树研发 |
| 文化 | 文明特有文化特征 |
| 经济 | 资源采集与分配 |
| 军事 | 军队/战争 |

**涌现程度**: 中高（文明级别的涌现行为）

#### 可视化

- 2D 六角格地图
- 文明边界/城市/军队标记
- 事件日志

#### 优势 & 我们可以借鉴

| 优势 | 借鉴 |
|---|---|
| 科技树/政策树设计 | 技能树设计参考 |
| 文明间自然竞争 | NPC 经济体之间的竞争逻辑 |
| 事件驱动叙事 | 世界事件的 GraphRAG 记录 |
| 长时间尺度的演化 | 法律/经济的长期演化设计 |

#### 劣势 & 我们如何超越

| 劣势 | AI Town 方案 |
|---|---|
| 无个体 Agent，无"人"的感觉 | **每个角色独立 Agent** |
| 宏观尺度，缺微观生活 | **个体生活模拟（吃饭/打工/恋爱）** |
| 回合制，缺乏实时感 | **实时制（可暂停）** |
| 无玩家角色扮演 | **完整的角色创建+人机协作** |
| 无个体关系/社交 | **好感度+婚姻+子嗣** |
| 浏览器端 | **Godot 4 + Steam** |

### 3.4 Project Sid

#### 项目定位

Project Sid 是一款游戏化的多 Agent 社会模拟，聚焦于大量 AI agent 在共同世界中的交互与协作。强调 agent 之间的**自组织行为**和**社会结构涌现**。

#### 技术架构

- 引擎: Unity（推测）
- Agent: 大量独立 Agent，每个有基础需求和目标
- LLM: 驱动 Agent 决策

#### 世界设计

- 地图: 3D 环境
- Agent: 数十到数百个
- 时间: 实时推进

#### 角色设计

- Agent 有基础需求（生存、社交、探索）
- 有物品/建造系统
- 自定义程度: 中等（开发者配置）

#### 社会模拟

- Agent 自发组织任务协作
- 资源竞争与合作
- 社会角色自然分化
- **涌现程度**: 高（大规模 Agent 群体行为）

#### 可视化

- 3D 渲染，视觉效果较好
- Agent 行为可观测

#### 优势 & 我们可以借鉴

| 优势 | 借鉴 |
|---|---|
| 大规模 Agent 并行模拟 | NPC 心跳的批量决策架构 |
| 社会角色自然分化 | 职业体系的多样性设计 |
| 物品/资源系统 | 经济系统的物品流转 |
| Agent 自组织行为 | NPC 自发事件（如集市形成） |

#### 劣势 & 我们如何超越

| 劣势 | AI Town 方案 |
|---|---|
| Agent 决策深度可能不够 | **分层记忆+反思+语义记忆** |
| 非开放产品 | **Steam 公开发布** |
| 无 GraphRAG 关系网络 | **五类实体知识图谱** |
| 无个体生命周期 | **完整寿命+继承系统** |
| 3D 门槛高 | **2D RPG Maker 风格更易上手** |

## 5. 核心差异化总结

### 我们独特的东西（竞品都没有）

1. **分层记忆 + 遗忘曲线**: 最接近人类记忆模型的 Agent 认知架构，不是简单的记忆流
2. **GraphRAG 五类实体图谱**: 世界中的角色/地点/组织/事件/物品互联互通，Agent 有"世界知识"
3. **Prompt 创建 + 人机协作**: 玩家不是旁观者，是共同创作者
4. **完整生命周期**: 从出生到死亡 + 子嗣继承 + 成就积分，形成一个有意义的循环
5. **Agent 意识透明化**: 三层面板让 AI 的思考不再黑盒
6. **法律演化**: NPC 宪法 + 玩家议会，社会规则可以演化
7. **Steam 原生可分发**: 不是 Demo 不是论文，是可购买可玩的完整产品

### 我们借鉴的最佳实践

| 来源 | 借鉴内容 |
|---|---|
| Generative Agents | 记忆流+检索三因子+反思+规划循环 |
| AI Town (a16z) | 像素俯视风格、对话日志可视化 |
| AIvilization | 技能树/科技树设计、文明级竞争逻辑 |
| Project Sid | 大规模 Agent 并行、自组织行为 |
