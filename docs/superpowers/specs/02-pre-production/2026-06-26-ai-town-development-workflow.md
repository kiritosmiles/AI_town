## [2] 预生产 (Pre-Production) 🔲 当前待进入

> **目标**: 把"要做什么"变成"怎么做"，所有人都能看同一份文档开始干活。

### 2.1 游戏设计文档 (GDD — Game Design Document)

PRD 说"要做什么"，GDD 说"怎么做"。GDD 是把 PRD 的每一条功能转化为**可实现的设计细节**。

| # | GDD 章节 | 对应 PRD | 需产出内容 |
|---|---|---|---|
| 1 | 游戏概述与愿景 | §1 | 一句话核心体验、游戏循环图 |
| 2 | 角色系统详细设计 | §5.1-5.3, P0-1~3 | 完整属性公式、性格参数模型、Tool 列表(名称/参数/返回)、角色创建 UI 流程图 |
| 3 | Agent 认知架构详细设计 | §5.2-5.4, P0-4~6 | LLM System Prompt 模板、记忆存储结构、遗忘曲线参数表、GraphRAG Schema (实体/关系类型定义)、反思触发条件表 |
| 4 | 世界与地图设计 | §4.2, P0-7~9 | Tile 类型表(地形/属性/通行)、Plot 划分规则、初始小镇布局图(俯视)、迷雾系统规则、分块加载策略 |
| 5 | 经济系统详细设计 | §5.6, P1-10~12 | 完整物品表(名称/价格/来源/用途)、货币产出曲线、材料配方树、排行榜算法 |
| 6 | 建造系统详细设计 | §5.6, P1-13 | 完整蓝图列表(需求/工期/产出)、Plot 购买规则、建筑生命周期状态机 |
| 7 | 生命与死亡系统设计 | §5.7, P1-14, P2-4~6 | 寿命计算公式(完整参数)、疾病列表(概率/影响/治疗)、成就积分算法、继承规则细节 |
| 8 | NPC 系统详细设计 | §5.5, P0-12, P1-7 | 初始 NPC 名册(200人:姓名/职业/背景/关系)、作息表模板、心跳调度算法、NPC 生成规则 |
| 9 | 社交系统详细设计 | §5.3, P2-1~3 | 好感度变化表(行为→数值)、关系阶段状态机、婚姻条件与流程、子嗣属性遗传算法 |
| 10 | 法律系统详细设计 | §5.5, P2-9~12 | 初始法律条文、犯罪-惩罚映射表、议会投票算法、法律提案流程 |
| 11 | 时间系统详细设计 | §4.3, P0-10 | 时间换算公式、游戏内日历(月/季节)、暂停与存档逻辑 |
| 12 | UI/UX 设计 | §3, P0-13~14, P1-15~16 | 完整 UI 布局图(每个面板)、交互流程图、信息架构 |

### 2.2 美术风格定义

> **这是 PRD 阶段后最需要补充的内容之一。**

| # | 需要提供 | 形式 | 优先级 |
|---|---|---|---|
| 1 | **参考游戏截图** | RPG Maker 风格的游戏截图 (俯视 Tile 地图参考) | 🔴 高 |
| 2 | **Tile 美术风格参考** | 地形(草/路/水/山/沙)的像素风格样式，色彩板 | 🔴 高 |
| 3 | **角色精灵风格参考** | 像素角色大小(如 32×32 / 48×48)、4 方向行走帧样式 | 🔴 高 |
| 4 | **建筑精灵风格参考** | 建筑在 Tile 上的呈现方式、大小比例 | 🟡 中 |
| 5 | **UI 风格参考** | 面板边框风格、字体选择、配色方案 | 🟡 中 |
| 6 | **色彩板 (Color Palette)** | 整体色调、季节变体色调 | 🟡 中 |
| 7 | **概念图 (Optional)** | 小镇全景概念图 | 🟢 低 |

**推荐参考来源**:
- RPG Maker MV/MZ RTP 素材 (`RPG Maker MV` 或 `RPG Maker MZ` 自带素材)
- 星露谷物语 (Stardew Valley) — 像素农田/小镇
- 宝可梦 心金/魂银 — 日式小镇俯视风格
- 《Omori》— 现代像素 RPG 美术
- 《Eastward》— 高品质像素风
- itch.io 免费像素 Tile 素材包 (搜索 `pixel art tilemap`)

### 2.3 音频方向

| # | 需要提供 | 形式 | 优先级 |
|---|---|---|---|
| 1 | BGM 风格参考 | 2-3 首参考曲目 (如星露谷物语原声、动物森友会) | 🟡 中 |
| 2 | 音效风格参考 | 脚步声/开门/UI 点击的音效参考 | 🟢 低 |
| 3 | 音频预算 | 原创音乐 或 素材包采购 或 CC0 免费素材 | 🟡 中 |

### 2.4 数值策划

| # | 内容 | 形式 | 优先级 |
|---|---|---|---|
| 1 | 经济数值表 | Excel/Google Sheet: 工资水平、物价、地产价格、税率、银行利率 | 🔴 高 |
| 2 | 角色属性数值 | 健康/体力/饥饿/心情的消耗速率与恢复速率 | 🔴 高 |
| 3 | 技能升级曲线 | 每级所需经验值、成功率、效果增益 | 🟡 中 |
| 4 | 建造经济平衡 | 每类建筑的 ROI (投资回报周期)、材料消耗量 | 🟡 中 |
| 5 | 好感度数值 | 每种交互行为的好感度增减量 | 🟡 中 |
| 6 | 排行榜数值 | NPC 富豪资产分布曲线、玩家追赶曲线 | 🟢 低 |

### 2.5 AI / Agent 行为设计

| # | 内容 | 形式 | 优先级 |
|---|---|---|---|
| 1 | System Prompt 模板 | 玩家 Agent 和 NPC Agent 的基础 System Prompt (角色描述+约束+可用 Tool 列表) | 🔴 高 |
| 2 | Tool 完整清单 | 每个 Tool: 名称、描述、参数 JSON Schema、返回格式、权限级别 | 🔴 高 |
| 3 | Agent 决策 Prompt 模板 | 感知摘要→记忆检索→LLM 推理→Tool 选择的完整 prompt chain | 🔴 高 |
| 4 | 反思 Prompt 模板 | 触发反思时的 prompt: "基于以下 N 条新记忆，总结洞察" | 🟡 中 |
| 5 | 重要性评分 Prompt | 对新记忆自动评分 1-10 的 prompt | 🟡 中 |
| 6 | NPC 初始 Prompt | 每个预设 NPC 的角色描述 (背景/性格/目标) — 至少 200 人 | 🟡 中 |

## [3] 原型验证 (Prototype) 🔲

> **目标**: 一个可玩的 MVP Demo，验证核心循环是否好玩。

### 3.1 原型范围 (对应 PRD P0)

| # | 功能 | 验收标准 |
|---|---|---|
| 1 | 角色 Prompt 创建 | 能创建角色，进入世界 |
| 2 | 基础 Tile 地图 | 预设小镇可走、可看 |
| 3 | Agent 基础决策 | 角色能自主移动+跟 NPC 对话 |
| 4 | 情景记忆 | NPC 能记住上次对话内容 |
| 5 | Agent 意识面板 | 玩家能看到 Agent 在想什么 |
| 6 | C 模式 NPC 心跳 | NPC 按作息表移动 |
| 7 | 存档/读档 | 经典模式可存档 |

### 3.2 原型验证标准

| 问题 | 通过标准 |
|---|---|
| Agent 决策延迟 | P50 < 10 秒 |
| NPC 记忆一致性 | 连续 3 次对话不"失忆" |
| 探索乐趣 | 内部试玩 5 人：有"想继续玩"的感觉 |
| 性能 | Godot 60 FPS, Python 进程 < 2GB RAM |
