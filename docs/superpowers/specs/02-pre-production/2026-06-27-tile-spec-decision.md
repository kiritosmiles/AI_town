# AI Town — Tile 规格决策记录

> 日期: 2026-06-27 | 状态: ✅ 已确定 | 决策人: 单人项目

---

## 决策结果

| 参数 | 决策值 | 理由 |
|---|---|---|
| **Tile 基础大小** | **32×32 px** | RPG Maker 黄金标准，像素素材生态最丰富，Godot 4 Tilemap 原生支持最佳 |
| **屏幕基础分辨率** | **1280×720 (HD)** | 现代显示器最低标准，比 RPG Maker 默认 816×624 更适配 Steam |
| **屏幕视野 (Tile 格数)** | **40×22.5 格** | 1280/32=40 列, 720/32=22.5 行，取 40×22（地图区）+ 右侧 UI 面板 |
| **角色碰撞框** | **单格 (32×32)** | 像素 RPG 标准，简化寻路和碰撞检测 |
| **角色精灵尺寸** | **48×64 px（占 1.5×2 格）** | 比单格稍大，视觉更突出，头部有表现空间；RPG Maker MV/MZ 标准尺寸 |
| **建筑占格** | **最小 2×2，最大 10×10** | 小屋到大型公建的跨度，对齐 Plot 系统 |
| **Plot 基准单元** | **10×10 Tile (320×320 px)** | 一栋小住宅+小院的标准地块，整数对齐 Tile |
| **地图总尺寸** | **1000×1000 Tile** | PRD 已定，物理尺寸 = 32,000×32,000 px |
| **分块 (Chunk) 大小** | **50×50 Tile (1600×1600 px)** | 1000/50=20×20=400 个 Chunk，每个约含 ~100-500 个 Tile，合理渲染+加载粒度 |
| **视野加载范围** | **3×3 Chunk (150×150 Tile)** | 玩家周围 9 个 Chunk 常驻渲染，其余仅存数据 |

## 影响范围

此决策直接影响：

| 受影响项 | 影响 |
|---|---|
| **T-4 美术参考** | 按 32×32 Tile + 48×64 角色收集参考 |
| **T-8 小镇布局** | 按 10×10 Plot、40×22 视野设计 |
| **T-13/14/15 素材制作** | 所有精灵按此规格制作 |
| **S3 Tilemap 性能测试** | 按 50×50 Chunk、3×3 加载范围测试 |
| **UI 设计 (T-10)** | 右侧面板宽度 = 1280 - 40×32 = 1280-1280×0 按比例分配 |

## 屏幕布局（规划）

```
┌──────────────────────────────────┬──────────────┐
│                                  │  Agent 面板   │
│      游戏地图区                    │  (320px)     │
│      960×720 px                  │              │
│      30×22.5 Tile                │  右侧 UI     │
│                                  │              │
│                                  │              │
└──────────────────────────────────┴──────────────┘
        1280×720 总分辨率
```

- **地图区**: 960×720 px = 30×22.5 Tile（地图滚动时视口内容）
- **右侧 UI**: 320×720 px，容纳 Agent 面板 / 背包 / 经济数据等

## 备选方案（已否决）

| 方案 | Tile | 否决原因 |
|---|---|---|
| 16×16 px | 更细粒度 | 像素太小看不清，美术工作量翻倍，1000×1000 地图更大 |
| 48×48 px | 更粗犷 | 像素素材生态少，RPG Maker 兼容差，角色细节反而难表现 |
